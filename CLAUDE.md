### Quick Local Build Test

Test the locally built RDKit with template-based coordinate generation and visualization:

```bash
test_local_rdkit "c1ccccc1"                             # benzene
test_local_rdkit "c1ccc2c(c1)ccc1ccccc12"              # anthracene (matches template)
```

This workflow:
1. Generates 2D coordinates using locally built RDKit with templates enabled
2. Displays the molecule image inline in iTerm2 (using Schrodinger sketcher)
3. **Important**: Coordinates are generated by local RDKit, then visualized without regeneration (avoids library conflicts)

**Example output:**
```

✓ Generated CXSMILES: c1ccccc1 |(-0.75,-1.29904,;0.75,-1.29904,;1.5,0,;0.75,1.29904,;-0.75,1.29904,;-1.5,0,)|

Step 2: Displaying molecule (preserving local RDKit coordinates)...
Molecule image saved to: /tmp/xxx.png
```

## 2D Coordinate Generation (Depictor)

Location: `Code/GraphMol/Depictor/`

### Key Files

- **RDDepictor.h/.cpp**: Main entry point for 2D coordinate generation
  - `compute2DCoords()` - Primary function for generating 2D coordinates
  - `Compute2DCoordParameters` - Parameters struct controlling generation behavior
  - `setRingSystemTemplates()` / `addRingSystemTemplates()` - Manage custom templates

- **Templates.h/.cpp**: Ring system template handling
  - `CoordinateTemplates` class - Singleton that manages ring templates
  - Templates are stored by atom count for fast lookup
  - `loadDefaultTemplates()` - Loads built-in ring system templates

- **TemplateSmarts.h**: Contains default ring system templates
  - Templates defined in SMARTS format (current branch work)
  - Each template includes 2D coordinates as CXSMILES

- **EmbeddedFrag.h/.cpp**: Managing already-embedded atoms and fragments
  - `EmbeddedAtom` - Represents an atom with 2D coordinates already assigned
  - `EmbeddedFrag` - Represents a fragment that has been embedded
  - **`matchToTemplate()`** - Core template matching function (EmbeddedFrag.cpp:455)
    - Loops through templates from `CoordinateTemplates::getMatchingTemplates()`
    - Performs substructure matching between ring system and templates
    - Checks stereochemistry compatibility
    - Copies coordinates from matched template to embedded fragment
    - **DEBUG OUTPUT**: Currently prints `TEMPLATE_MATCH: <SMILES>` to stderr when a template matches

- **DepictUtils.h/.cpp**: Utility functions for 2D embedding
  - `embedRing()` - Embed a ring as a convex polygon
  - Various geometric helper functions

### Tests

- **catch_tests.cpp**: Main Catch2 test suite
- **testDepictor.cpp**: Additional depictor tests
- **test_data/**: Test molecules and reference data

### Current Work (SMARTS_templates branch)

Working on accepting ring templates in SMARTS format instead of SMILES. Key changes involve parsing templates as SMARTS patterns for more flexible template matching.

## RDKit-Specific Constants and Patterns

### Common Constants in Depictor Code
- `RDDepict::BOND_LEN` - Standard 2D bond length (typically 1.5)
- `NUM_BONDS_FLIPS` - Maximum number of bond flips during collision resolution
- `MAX_COLL_ITERS` - Maximum collision resolution iterations
- `COLLISION_THRES` - Collision detection threshold
- `BOND_THRES` - Bond threshold for detection

### Common Helper Functions
- `_recurseAtomOneSide()` - Recursively partition atoms on one side of a bond/spiro center
- `getRotatableBonds()` - Find rotatable bonds between two atoms
- `findCollisions()` - Detect atomic collisions in 2D coordinates
- `totalDensity()` - Calculate coordinate density for optimization
